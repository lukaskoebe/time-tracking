# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

All commands run from `time-tracking/` (the inner project directory):

```bash
pnpm dev          # Start dev server on port 3000
pnpm build        # Production build
pnpm preview      # Preview production build
pnpm test         # Run tests with Vitest
pnpm lint         # ESLint
pnpm typecheck    # TypeScript type check (no emit)
pnpm format       # Prettier formatting
pnpm db:push      # Push schema changes directly to DB (dev)
pnpm db:generate  # Generate migration files
pnpm db:migrate   # Run pending migrations
pnpm db:studio    # Open Drizzle Studio
```

## Environment

Copy `.env` and set:
- `DATABASE_URL` — PostgreSQL connection string
- `BETTER_AUTH_SECRET` — min 32-char secret for better-auth

## Architecture

**Stack:** TanStack Start (React 19 + Vite + Nitro) with file-based routing, shadcn/ui, Tailwind CSS v4, Drizzle ORM + PostgreSQL, better-auth.

**Routing:**
- Routes live in `src/routes/` — filenames become URL paths automatically. `__root.tsx` is the HTML document shell.
- `src/routeTree.gen.ts` is auto-generated by TanStack Router — never edit manually.
- `_authenticated.tsx` is a pathless layout route — all routes in `_authenticated/` are protected and get the sidebar layout.
- API server routes (raw HTTP handlers) use `createFileRoute` with a `server.handlers` property (GET/POST/etc). See `src/routes/api/auth/$.ts`.

**Server functions:**
- Data fetching and mutations use `createServerFn` from `@tanstack/react-start` (not Next.js server actions).
- Server functions that need auth call `getRequest()` from `@tanstack/react-start/server` then `auth.api.getSession({ headers: request.headers })`.
- Live in `src/server/` — suffix `.server.ts` for pure DB helpers, `.functions.ts` or plain `.ts` for `createServerFn` wrappers.
- After mutations, call `router.invalidate()` to re-run loaders.

**Auth (better-auth):**
- Server config: `src/lib/auth.ts` — uses Drizzle adapter + `tanstackStartCookies()` plugin (must be last).
- Client: `src/lib/auth-client.ts` — `signIn`, `signUp`, `signOut` exports.
- Session helper: `src/lib/auth.functions.ts` — `getSession` server function used in `beforeLoad`.
- Auth API handler: `src/routes/api/auth/$.ts`.

**Database (Drizzle + pg):**
- Schema: `src/lib/db/schema.ts` — includes better-auth tables (`user`, `session`, `account`, `verification`) and app tables (`project`, `time_entry`).
- DB client: `src/lib/db/index.ts` — `drizzle(pool, { schema })` enables relational queries via `db.query.*`.

**UI:**
- UI components follow shadcn/ui patterns in `src/components/ui/`. Add with `pnpm dlx shadcn@latest add <component>`.
- Path alias `@/` maps to `src/`.
- `cn()` from `src/lib/utils.ts` merges Tailwind classes. `formatDuration`, `formatTotalTime`, `getDurationSeconds`, `PROJECT_COLORS` are also exported from there.
- Tailwind CSS v4 with oklch color variables in `src/styles.css`. Dark mode via `.dark` class.
- Toaster (`sonner`) is mounted in `__root.tsx`. Use `toast.success/error()` anywhere.
